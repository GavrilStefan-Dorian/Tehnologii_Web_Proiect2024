HeMa (Herbal Web Manager)

Autori: Pintecan Alexandru, Ezaru Tudor 2B1

1. Introducere

1.1. Scop

Documentul prezinta cerinele software pentru HeMa (Herbal Web Manager), o aplicatie web de tip ierbar digital, care permite organizarea si gestionarea colectiilor de reprezentari ale plantelor si caracteristicile acestora, dar si intocmirea anumitor statistici pe baza plantelor existente.

1.2. Conventii ale documentului

In cadrul proiectului, implementarea a fost realizata cu documente HTML pentru generarea paginilor web si documente CSS pentru stilizarea acestora.

Elementele de design folosite sunt:

  • Diferite stiluri de font: Arial, Tahoma, sans-serif
  • Evidentierea textului prin boldare
  • Folosirea imaginilor pentru decorarea containerelor
  • Efecte vizuale la butoane si imagini

1.3. Publicul tinta

Din publicul tinta pentru aceasta documetatie pot face parte:

  • Dezvoltatori
  • Manageri de Proiect
  • Utilizatori
  • Testeri

1.4. Scopul aplicatiei

Scopul aplicatiei HeMa (Herbal Web Manager) este furnizarea unei platforme intuitive pentru gestionarea si explorarea unor colectii de imagini reprezentand plante presate, impreuna cu atributele lor asociate. Prin intermediul facilitatilor avansate de cautare, utilizatorii pot naviga rapid si eficient prin diversele colectii de imagini.

De asemenea, aplicatia ofera utilizatorilor posibilitatea de a vizualiza clasamentul plantelor din colectiile existente, de a studia statistici despre acestea pe baza mai multor criterii si de a genera diverse statistici despre colectiile lor de imagini sau despre colectiile deja existente.

2. Descriere generala

2.1. Perspectiva produsului

HeMa (Herbal Web Manager) este o aplicatie realizata in scop academic care serveste drept o platforma pentru pasionati si cercetatori in botanica. Astfel, prin intermediul acestei aplicatii, utilizatorii isi pot completa/imbogati colectiile personale de plante virtuale, obtinandu-se astfel un cerc de interes comun pentru comunitatea pasionata de lumea vie.

2.2. Functiile produsului

In cadrul produsului, clientii vor gasi diferite functionalitati, printre care se numara:vizualizarea colectiilor deja existente, intocmirea de statistici avand in vedere anumite criterii, vizualizarea unui clasament al celor mai populare plante din produs, posibilitatea cautarii avansate, pe baza unor criterii bine definite, generarea albumelor proprii de catre utilizator.

  • Vizualizarea colectiilor: utilizatorii pot parcurge plantele listate deja in colectiile existente
  • Intocmirea de statistici: utilizatorii pot genera statistici diverse despre colectia lor de imagini, sau despre colectiile deja existente, disponibile in formate deschise, cum ar fi CSV si PDF
  • Clasament al plantelor populare: aplicatia genereaza si afiseaza un clasament al celor mai populare plante colectate, plantele care sunt in centrul atentiei fiind astfel mai vizibile
  • Cautare avansata bazata pe anumite criterii: utilizatorii pot cauta anumite plante dupa diverse criterii, precum climat, mediu de viata, culoare
  • Generare de albume: utilizatorii isi pot creea albumele proprii, pe ce criterii doresc acestia

2.3. Clasele de utilizatori si caracteristicile lor

Utilizatorii nostri pot face parte din una din urmatoarele clase, dar nu numai. Astfel, iata cateva clase care pot utiliza produsul:

  • Elevi/Studenti:pot folosi aplicatia in scop educational(ex: compararea caracteristicilor a 2 plante, aflarea de informatii despre o anumita planta/specie etc.)
  • Profesori:poate folosi aplicatia pentru a creea anumite materiale didactice sau pentru a expune elevilor la ora anumite diferente intre plante/specii
  • Cercetatori:pot adauga caracteristici unei plante deja listate/pot adauga o noua planta in urma unor studii efectuate de acestia
  • Pasionati de lumea vie:pot adauga diverse poze cu plante pozate de acestia in diversele lor activitati(ex:drumetiile pe munte, plimbari prin paduri etc.)
  • Utilizatori ocazionali:pot vizita aplicatia noastra pentru a afla o infomatie rapida din lumea plantelor
  • Administrator: gestioneaza si rezolva problemele aplicatiei web, poate da rolul altor utilizatori pentru a deveni administartori, controland astfel colectiile, statisticile si clasamentul afisat in aplicatie

2.4. Mediul de operare

Aplicatia va functiona pe toate motoarele de cautare majore, fiind proiectata sa fie responsive atat pentru platformele desktop cat si pentru cele mobile.

2.5. Constrangeri de proiectare si implementare

Constrangerile pe care produsul le poate intampina sunt:

  • Diversitatea lingvistica: aplicatia este construita avand suport doar in limba romana, alte limbi existente nefiind suportate
  • Politicile de cofidentialitate: produsul acceseaza API-ul public Unsplash pentru preluarea de imagini

2.6. Documentatia utilizatorului

Utilizatorul este documentat si ghidat prin intermediul paginilor de "Ajutor" si "Despre noi", despre cum poate acesta sa foloseasca produsul.

2.7. Dependinte

Pentru utilizarea aplicatiei HeMa (Herbal Web Manager), clientii trebuie sa indeplineasca cateva cerinte minimale. Printre acestea se numara:

  • Conexiunea stabila si continua la o conexiune wireless
  • Accesul continuu la API Unsplash pentru accesarea imaginilor relevante
  • Accesul la o resursa (PC, laptop, tableta, telefon) care suporta un browser si conexiunea la internet

3. Cerinte ale interfetelor externe

3.1. Interfete utilizator

Interfata utilizator a HeMa este conceputa pentru a fi intuitiva si accesibila, permitand utilizatorilor sa navigheze si sa interactioneze eficient cu aplicatia. Caracteristicile interfetei dintre utilizator si aplicatie sunt:

  • Existenta mai multor pagini: utilizatorului i se permite sa acceseze functionalitatile aplicatiei, parcurgand mai multe pagini.
  • Standarde GUI: designul aplicatiei HeMa (Herbal Web Manager) este unul modern, creeand un mediu placut pentru utilizator.
  • Structura vizuala a aplicatiei: prezenta barelor de cautare si a butoanelor sunt plasate astfel incat utilizatorului sa ii fie usor sa le foloseasca.
  • Butoane: utilizatorul va putea folosi functionalitatile aplicatiei prin intermediul unor butoane.
  • Pagini de logare si inregistrare: pentru a accesa functionalitatile aplicatiei, userul va trebui sa isi creeze un cont prin intermediul paginii de inregistrare si accesarea contului folosind pagina de autentificare.
  • Report a bug: utilizatorii pot prezenta bug-uri si sugestii pentru imbunatatirea experientei in cadrul aplicatiei.

3.2. Interfete hardware

HeMa este conceputa special pentru a se utiliza pe browserele populare. Componenta hardware nu are o relevanta semnificativa, ci doar dispozitivul sa fie capabil sa se conecteze la internet si sa suporte un browser.

3.3. Interfete software

API Unsplash se foloseste pentru a putea accesa si afisa paginile. Interfata software mai are un rol semnificativ in compatibiliatea cu browserele web.

3.4. Interfete de comunicare

Aplicatia foloseste protocoalele HTTP si HTTPS pentru transferul datelor.

5. Alte functionalitati nonfunctionale

5.1. Cerinte de performanta

Aplicatia poate sa suporte mai multi utilizatori simultan.

5.2. Cerinte de siguranta

Utilizatorii vor avea parole care vor fi prelucrate intr-un hash pentru a evita pericolul de spargere a conturilor.

5.3. Atribute de calitate software

Aplicatia va avea mentenanta pe termen lung si utilizatorii vor avea asigurat accesul la features.

5.4. Reguli de afaceri
